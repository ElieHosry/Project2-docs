Terminology Guide

Unified terms and symbols used across all module documents. When writing new material, use these exact abbreviations.

Module Terms

Term Meaning
T1, T2 Trimester 1, Trimester 2
M1 Milestone 1 — week 9. Phase per track: CS Core technology base · Design/Audio/UI Research · Art Research, early concept ideation and mood boards
M2 Milestone 2 — week 13. Phase per track: CS Core technology showcase · Design/Audio/UI Iterated Prototype · Art Style Guide and Art direction
Goal IDs — G / D / A / TA / TU Every goal carries a track-prefixed pool ID: G = CS, D = Design, A = Art, TA = TEAM Audio, TU = TEAM UI. Numbers identify goals within a pool; both milestones share the numbering (M1 D2 and M2 D2 are different goals of the same pool slot) — write "M1 G5" when the milestone isn't clear from context
G1–G3 ⭐ The CS common goals, identical across all project categories, always selected; each CS student's milestone = 1 goal, so a team's 6 CS goals are G1–G3 ⭐ + 3 picks from G4+ (instructor approval)
Goal A heavy component owned by one student, broken into sub-tasks, ~40 hours per milestone (CS); Design/Art goals are deliverable-sized (2 ≈ one CS goal)
Goal pool Candidate goals per milestone per category/track. CS pools start at 16 (GovTech has G17); Design D1–D11; Art A1–A7; TEAM TA1/TU1. All pools are extensible — instructor-approved goals from new proposals are appended (G18…, D12…, TA2…, TU2…)
D1 ⭐ The Design track's first common goal (per milestone); each Design student takes up to 3 Design goals per milestone — commons D1 ⭐ + TU1 ⭐ plus picks from D2+
TU1 ⭐ The Design track's UI common goal (one per milestone; migrated from TEAM, July 15) — graded by the UI instructor under the Design track; affects Design students only
A1 ⭐ The Art track's common goal (per milestone); each Art student takes 2 Art goals per milestone (teams have 0–2 artists), first artist A1 ⭐ + 1 pick from A2+
Quality ladder Base → Good → Exceptional rungs on Design and Art goals; Base = "Done", higher rungs raise the grade, not the deadline
Done / Partially Done / Missing The goal grading scale, identical for all tracks (CS, Design, Art, TEAM); "Untested" is the pre-grading default in workbooks
TEAM goal / TA1 ⭐ The cross-disciplinary Audio goal, mandatory for every team (TA1 ⭐ in each milestone); shared grade (affects every member), coordinated by a champion, graded by the Audio instructor; future optional picks would join as TA2+
Foundation sprint Weeks 4–5: project-agnostic head start (G1 skeleton, CI setup, schema drafts, spikes) before week-5 approval
Team due diligence Weekly instructor follow-up per team: goal progress, blockers, integration health
Backend scope How far a project's backend goes — see levels below and Appendix A
Local only Local backend suffices; a custom server adds little
Server-ready (T2) Works locally in T1; clean seam to lift the C++ service onto the Linux server in T2
Server-indicated (T2) The design inherently needs a shared server; local/Firebase is a T1 stopgap
Purple requirement The newly suggested core tech requirement added per category (last matrix row, purple text)

Symbols

Symbol Meaning

❌ Not needed / rarely applicable
⏭️ Deferred to second trimester per module plan
⭐ Common/mandatory goal (G1–G3, D1, A1, TA1, TU1)
