Terminology Guide

Unified terms and symbols used across all module documents. When writing new material, use these exact abbreviations.

Module Terms

Term Meaning
T1, T2 Trimester 1, Trimester 2
M1 Milestone 1 — week 9, Core technology base phase
M2 Milestone 2 — week 13, Core technology showcase phase
G1…G6 The 6 goals of a milestone, one per student
G1–G3 ⭐ The common goals, identical across all project categories, always selected
G4–G6 The customizable goals, chosen from the category's pool (instructor approval)
Goal A heavy component owned by one student, broken into sub-tasks, ~40 hours per milestone
Goal pool / #N The candidate goals per milestone per category (16 at minimum — pools grow as approved proposals add goals at the end); #N refers to a goal's number in that pool
Foundation sprint Weeks 4–5: project-agnostic head start (G1 skeleton, CI setup, schema drafts, spikes) before week-5 approval
Team due diligence Weekly instructor follow-up per team: goal progress, blockers, integration health
Backend scope How far a project's backend goes — see levels below and Appendix A
Local only Local backend suffices; a custom server adds little
Server-ready (T2) Works locally in T1; clean seam to lift the C++ service onto the Linux server in T2
Server-indicated (T2) The design inherently needs a shared server; local/Firebase is a T1 stopgap
Purple requirement The newly suggested core tech requirement added per category (last matrix row, purple text)

Symbols

Symbol Meaning
✅ Requirement applies directly
🔶 Applies with domain adaptation (reinterpreted meaning)
🔵 Optional — only if the project design needs it
❌ Not needed / rarely applicable
⏭️ Deferred to second trimester per module plan
⭐ Common goal (fixed for all categories)

Technical Acronyms

Acronym Meaning
BaaS Backend-as-a-Service (Firebase) — remote, but not a student-built server
CI Continuous Integration — automated build, test, packaging pipeline
DoD Definition of Done — completion criteria of a goal (includes integration into the main build)
FE / BE Front end / Back end
WASM WebAssembly — C++ compiled for the browser via Emscripten (emsdk)
GLSL OpenGL Shading Language
ECS Entity Component System — the data-driven object architecture of G2
FSM Finite-State Machine
RBAC Role-Based Access Control
DSP / FFT Digital Signal Processing / Fast Fourier Transform (audio analysis)
SIR Susceptible–Infected–Recovered epidemic model
CRDT / OT Conflict-free Replicated Data Type / Operational Transform (collaborative editing)
SLA Service-Level Agreement (workflow deadlines)
MSVC Microsoft Visual C++ compiler (Visual Studio)
ASan / UBSan Address / Undefined-Behavior Sanitizer (ASan only on MSVC; UBSan via clang-cl or WSL)
WSL Windows Subsystem for Linux
NDK Android Native Development Kit (C++ on Android)
