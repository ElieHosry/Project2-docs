CS Goals Pool per Category

(Formerly "Milestone Goals Pool" — renamed when the Design, Art and TEAM goal structures joined the module; the filename is unchanged so existing links keep working.)

A goal is a heavy component owned by one student, who breaks it into sub-tasks (~40 hours per milestone). Each milestone below offers a pool of at least 16 goals — 16 is the starting point; when an approved proposal introduces a new goal that fits the module criteria, it is appended to the end of that category's pool. The instructor selects 6 per milestone per team.

Goals G1–G3 of every pool are common to all project categories (marked ⭐) and are always selected — they generalize G1–G3 of the Game example. Goals G4 onward are the category-specific options. Teams may propose new goals, subject to instructor approval.

Milestone titles follow the example file: Milestone 1 (Core technology base phase) and Milestone 2 (Core technology showcase phase).

Every goal has an expandable Tech & dependencies block (click to open) listing the technology a student would use, plus prerequisite goals and the goals that depend on it. Baseline for all goals: C++17/20 core, CMake + Git, low-level libraries only (SDL2, OpenGL, ImGui, FMOD/OpenAL, FreeType…), web front-ends via HTML/JS/CSS/WASM, external services via Firebase. "G1/G2/G3" in prerequisites refer to this pool's common goals. Development happens on Windows (Visual Studio / VS Code); all listed libraries are cross-platform, and CI builds on both Windows and Linux — see Appendix E in the Appendices.

Common Goals (all categories)

M1: 1⭐ Application Shell, Real-time C++ Simulation Loop, Input & Core Logic · 2⭐ Data-Driven Architecture of Core Objects / Entity System · 3⭐ Debugging, Profiling, Stress Test & CI

M2: 1⭐ Architecture — Optimization & Platform Validation · 2⭐ Content Editor — Core · 3⭐ CI — CMake, Code Quality, Build Standards & Packaging

Games

Milestone 1 (Core technology base phase)

  • G1 ⭐ Application Shell, Real-time C++ Simulation Loop, Input & Core Logic
    Tech & dependencies
    • Tech: C++17/20, SDL2 (window, input, timing), fixed-timestep loop, CMake, Git

    • Depended on by: All other goals build on the shell and loop
  • G2 ⭐ Data-Driven Architecture of Core Objects / Entity System
    Tech & dependencies
    • Tech: C++ (templates, reflection patterns), JSON (nlohmann/json or RapidJSON), hot-reload file watching
    • Prerequisites: G1 (application shell & loop)

  • G3 ⭐ Debugging, Profiling, Stress Test & CI
    Tech & dependencies
    • Tech: ImGui debug overlays, profiler (Tracy / platform profiler), Catch2 or GoogleTest, GitHub Actions or GitLab CI (Windows + Linux build matrix)
    • Prerequisites: G1 (application shell & loop)
    • Depended on by: M2 G3 (CI advanced); all goals rely on the test/CI safety net
  • G4 2D Graphics — sprite/tile renderer, camera, batching
    Tech & dependencies
    • Tech: SDL2 renderer or OpenGL ES, GLSL, FreeType (text), stb_image (textures)
    • Prerequisites: G1 (shell & loop)
    • Depended on by: UI/HUD, particles, charts, editors — every visual goal

    Tech & dependencies
    • Tech: C++ + GLM math, custom integrator, broadphase (uniform grid / quadtree)
    • Prerequisites: G1 (shell & loop)

  • G6 Audio System — playback, mixing, event triggers
    Tech & dependencies
    • Tech: FMOD or OpenAL, stb_vorbis / dr_wav decoding, event-triggered playback
    • Prerequisites: G1 (shell & loop)

  • G7 Procedural Level Generation — algorithms, validation, seeding

    • Prerequisites: M1 G2 (level data model)

  • G8 Enemy AI & Pathfinding — A*, steering, state machines
    Tech & dependencies
    • Tech: A* / flow fields, steering behaviors, finite-state machines (custom C++)

  • G9 Animation System — sprite animation, blending, state graph
