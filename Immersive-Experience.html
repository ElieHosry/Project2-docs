Immersive Experience

Scope

Atmosphere-first interactive experiences — exhibits, generative visualizers, interactive stories, ambient installations. Technically a near-twin of Games, but presentation, audio-visual craft and mood carry the product instead of game mechanics.

Examples

  1. Interactive museum exhibit — animated 2D storytelling scenes reacting to touch/motion input; scene editor for curators; audio system. References:

    • teamLab Borderless Tokyo
    • teamLab walkthrough (YouTube)

  2. Generative audio-visual experience — real-time music visualizer with user interaction, C++ DSP/particle simulation, preset editor. References:

    • Patatap (playable)
    • Patatap in use (YouTube)

  3. Virtual campus tour — explorable 2D/2.5D map with ambient audio and interactive hotspots; content editor to place hotspots and media. References:

    • YouVisit campus tours
    • YouVisit tour demo (YouTube)

  4. Interactive storybook — animated scenes with branching narrative, particle effects, and reactive sound; story/scene editor as the custom tool. References:

    • Florence (Annapurna Interactive)
    • Florence launch trailer (YouTube)

  5. Interactive planetarium explorer — pan/zoom night sky rendered from star-catalog data, constellation overlays, gesture navigation, ambient audio. References:

    • Stellarium Web (live)
    • How to use Stellarium Web (YouTube)
    • Star Walk 2 (App Store)

How each requirement applies

Common to all five: technically the closest category to Games — Simulation Loop ✅ (animation/particles/DSP per frame) · 2D ✅ (custom scene rendering) · C++ Performant Service (the experience engine) · Custom Tools ✅ (scene/content editors) · Data-Driven Architecture (scenes and content as data) · Application Communication · Continuous Integration · Front-end + Local Backend · Auth 🔵 optional (personalized experiences).

1. Interactive museum exhibit

Platform fit: Native mobile — the reference phone (Pixel 8a/9a) mounted as the exhibit kiosk (or cast to a large display); the PC build is used for development.
Front end: Fullscreen scene rendering, touch/motion input layer. Languages: C++, GLSL.
Back end: Scene & interaction engine, particle systems, spatial audio, Firebase content/stats. Languages: C++.
Backend scope: Local only.
  • Simulation loop: touch/motion-reactive animation and particle systems.
  • Custom tool: curator scene editor. Networking: optional multi-visitor shared interaction.
  • External DB / Analytics ⏭️: exhibit content, visitor interaction heatmaps.
  • Spatial/reactive audio: soundscapes that follow touch position and scene state across the installation.

2. Generative audio-visual visualizer

Platform fit: Native mobile — the phone microphone drives the visuals; AAudio on the reference phone (Pixel 8a/9a) gives low-latency input. PC build for development.
Front end: Particle visuals, preset/mapping GUI. Languages: C++, GLSL.
Back end: DSP engine (FFT, beat detection), audio I/O, generative parameter mapping. Languages: C++.
Backend scope: Local only.
  • Simulation loop: C++ DSP (FFT, beat detection) driving particle simulation per frame — the most performance-demanding example.
  • Custom tool: preset/mapping editor. External DB / Analytics ⏭️: shared presets, usage stats.
  • Spatial/reactive audio: the core of the project — analysis and reactive mixing ARE the experience.

3. Virtual campus tour

Platform fit: Web browser on mobile + Native mobile — prospective students tour from anywhere.
Front end: Map view with hotspots, media overlays, tour navigation. Languages: HTML/JS/CSS + C++ renderer via WASM, GLSL.
Back end: Navigation/zone engine, ambient audio zones, content services, Firebase. Languages: C++.
Backend scope: Local only.
  • Simulation loop: smooth map navigation, hotspot animation, ambient audio zones.
  • Custom tool: hotspot/media placement editor. 3D: optional 2.5D parallax views.
  • External DB / Analytics ⏭️: tour content, most-visited-location stats.
  • Spatial/reactive audio: ambient zones crossfade by map position — birds in the quad, chatter in the cafeteria.

4. Interactive storybook

Platform fit: Native mobile — bedtime-reading on the reference phone.
Front end: Animated scene rendering, touch interaction. Languages: C++, GLSL.
Back end: Narrative graph engine, reactive audio, reading-progress services. Languages: C++.
Backend scope: Local only.
  • Simulation loop: scene animation, particles, reactive sound.
  • Data-driven: branching narrative as a data graph — a strong showcase. Custom tool: story/scene editor with branch visualization.
  • External DB / Analytics ⏭️: stories, reading progress, branch-choice stats.
  • Spatial/reactive audio: sound effects tied to touch interactions and narrative state per scene.

5. Interactive planetarium explorer

Platform fit: Native mobile + Web browser on mobile — stargazing happens outdoors, phone in hand.
Front end: Sky renderer, constellation overlays, gesture controls. Languages: C++ via WASM + HTML/JS shell (web), native C++ (phone); GLSL.
Back end: Celestial position engine, star-catalog services, Firebase content delivery. Languages: C++.
Backend scope: Local only.
  • Simulation loop: smooth sky rendering with real-time celestial position math (time- and location-based).
  • Data-driven: star catalogs (e.g., HYG) and constellation data as content. Custom tool: label/content editor.
  • Input: gesture navigation (pan, pinch, inertia). Camera: smooth transitions between sky regions.
  • External DB / Analytics ⏭️: saved views, most-viewed objects; content delivery via Firebase.
  • Spatial/reactive audio: ambient soundscape shifting with sky region and time of night.

  • G1 ⭐ Application Shell, Real-time C++ Simulation Loop, Input & Core Logic

    • Prerequisites: G1 (shell & loop).

  • G3 ⭐ Debugging, Profiling, Stress Test & CI

  • #4 2D Scene Rendering & Animation
    Tech & dependencies
    • Tech: OpenGL scene renderer with GLSL, tween/sprite animation of exhibit artwork, FreeType captions, stb_image textures.
    • Prerequisites: G1 (loop); G2 (scene data to render).
    • Feeds into: M1 #5 (particles render through it); M2 #4 (lighting/shaders extend it).
  • #5 Particle & VFX System
    Tech & dependencies
    • Tech: OpenGL-instanced particles blooming from visitor touch points, emitter configs stored as JSON data.
    • Prerequisites: M1 #4 (renderer).
    • Feeds into: Touch feedback together with M1 #9; the visual wow-factor of the exhibit.
  • #9 Interaction / Trigger System
    Tech & dependencies
    • Tech: Spatial hit-testing over scene objects, touch triggers firing animation, particle and audio events per exhibit zone.
    • Prerequisites: G1 (touch input); M1 #4 (scene objects to hit-test).

    Tech & dependencies

    Tech & dependencies
    • Tech: OpenGL renderer for visualizer layers — backgrounds, geometric shapes, GLSL effects, FreeType text.

    • Feeds into: M1 #5 (particles composite over it); M2 #8 (generative visuals).

    Tech & dependencies
