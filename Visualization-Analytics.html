Visualization & Analytics

Scope

Products whose deliverable IS the visualization: ingesting real data and rendering it live with the team's own chart, map and plot engines — dashboards, explorers, analytics tools. Two guard-rails define the boundary: the data must move (streaming, animated playback or interactive re-aggregation — a static chart is a CRUD app), and the domain must not be claimed by another category (financial charting → Fintech; civic service dashboards with staff workflows → GovTech; simulation result views → Simulator; chart authoring tools → Creative Tools). What lives here: the uncovered slice — city/IoT metrics, logs and telemetry, personal/fitness data, scientific datasets, sports analytics.

Examples

  1. Live city metrics dashboard — real-time simulated utility feeds (water, power, waste) processed in C++; map-based live visualization with zone drill-down; dashboard composer as the custom tool. (Moved from GovTech, July 2026 — the visualization is the product; the staff-workflow side lives in GovTech.) References:
    • Snap4City control room dashboards
    • Snap4City dashboard demo (YouTube)
  2. Log & telemetry explorer — high-volume application/server log ingestion, time-series charts, live tailing, query & filter engine; the Grafana/Kibana product shape built on the team's own renderer. References:
    • Grafana OSS (official)
    • Kibana (Elastic, official)
  3. Personal fitness data explorer — import workout files (GPX/CSV), explore pace/heart-rate/elevation with linked charts and route maps, animated activity playback, trends over months of history. References:
    • Strava (official)
    • Strava features
  4. Scientific dataset plotter — load large numeric datasets (sensor arrays, experiment outputs), scatter/line/heat plots at 1M+ points via LOD, brushing & linked views, publication-quality export. References:
    • ParaView (Kitware, official)
    • ParaView features
  5. Sports match analytics visualizer — event-stream match data (passes, shots, positions) rendered as pitch maps, heatmaps and timelines with animated replay and coach annotations. References:
    • Hudl Statsbomb (official)
    • Hudl Sportscode

How each requirement applies

Common to all five: 2D ✅ (custom chart/map/plot rendering IS the product — no chart libraries) · C++ Performant Service (ingestion + aggregation engines) · Data-Driven Architecture ✅ (data sources, chart configs, dashboards as data) · Application Communication · Continuous Integration · Front-end + Local Backend.

1. Live city metrics dashboard

Platform fit: Web browser on mobile + Native PC (control-room ops) — citizens and officials view the live dashboard on phones; the control-room display stays on the PC side.
Front end: Map layers, live dashboard panels, alert displays. Languages: C++, GLSL (via WASM); HTML/JS/CSS shell.
Back end: Sensor feed engine, aggregation & statistics, dashboard configuration, Firebase history. Languages: C++.
Backend scope: Local now, server-ready (T2).
  • Simulation loop: C++ engine generates and processes simulated sensor feeds (water, power, waste) — high-frequency by construction.
  • 2D: map-based visualization with live utility layers and alert badges. Custom tool: dashboard composer — panels, layers and thresholds arranged without recompiling.
  • External DB / Analytics ⏭️: historical feed data, alert statistics. Auth ⏭️: 🔵 operator accounts. Networking: 🔵 live push to viewers.
  • Large-dataset rendering: a year of per-zone readings scrubbed on a timeline without dropping frames.

2. Log & telemetry explorer

Platform fit: Web browser on mobile + Native PC (analyst ops) — on-call engineers check dashboards and alerts on phones; the deep-dive desk stays on the PC side.
Front end: Time-series charts, live log tail, query bar, dashboard grid. Languages: C++, GLSL (via WASM); HTML/JS/CSS shell.
Back end: Log ingestion & parsing engine, time-series store, query/filter engine, Firebase saved views. Languages: C++.
Backend scope: Local now, server-ready (T2).
  • Simulation loop: continuous ingestion — thousands of log lines/second parsed, indexed and aggregated live (a bundled generator replays real log fixtures).
  • 2D: time-series rendering with smooth zoom from weeks to milliseconds. Custom tool: dashboard/query composer with saved views.
  • External DB / Analytics ⏭️: saved dashboards, usage stats. Auth ⏭️: 🔵. Networking: 🔵 remote log sources.
  • Large-dataset rendering: a million-line log rendered as density strips and percentile bands, interactive throughout.

3. Personal fitness data explorer

Platform fit: Native mobile + Web browser on mobile — your data on your phone (reference device); web build for the big-screen review. PC is the development vehicle.
Front end: Linked pace/HR/elevation charts, route map, activity playback, trends. Languages: C++, GLSL.
Back end: GPX/CSV import pipeline, aggregation engine (splits, zones, trends), Firebase sync. Languages: C++.
Backend scope: Local only.
  • Simulation loop (adapted): animated activity playback (the run replayed on the map with synced chart cursors) and live re-aggregation while brushing.
  • 2D: multi-series linked charts + route rendering from raw GPS points. Custom tool (adapted): personal dashboard/layout configurator.
  • External DB: activity sync across devices. Auth: ✅ personal data. Analytics ⏭️: feature usage.
  • Large-dataset rendering: years of workouts — hundreds of thousands of trackpoints — in one zoomable timeline.

4. Scientific dataset plotter

Platform fit: Web browser on mobile + Native PC (authoring) — results explored and shared from the phone; heavy datasets prepared on the PC side.
Front end: Scatter/line/heat plots, brushing & linked views, export preview. Languages: C++, GLSL (via WASM); HTML/JS/CSS shell.
Back end: Columnar dataset store, decimation/LOD engine, statistics, publication export. Languages: C++.
Backend scope: Local only.
  • Simulation loop (adapted): interactive render loop — re-decimation and re-binning at every pan/zoom/brush, animated dataset sweeps.
  • 2D: the plot engine IS the product — axes, ticks, millions of points, FreeType labels. Custom tool (adapted): plot/figure composer with styles.
  • External DB / Analytics ⏭️: shared figure library. Auth ⏭️: 🔵. 3D: 🔵 surface plots for strong students.
  • Large-dataset rendering: the flagship — 1M+ points at 60 fps on the dev machine, LOD-decimated gracefully on the reference phone.

5. Sports match analytics visualizer

Platform fit: Native mobile + Web browser on mobile — match review on the phone or tablet-free touchline; web build for shared session links. PC is the development vehicle.
Front end: Pitch map, heatmaps, event timelines, animated replay, annotation layer. Languages: C++, GLSL.
Back end: Match event-stream engine, per-player aggregation, annotation store, Firebase team library. Languages: C++.
Backend scope: Local now, server-ready (T2).
  • Simulation loop (adapted): animated match replay — event streams played back with live-updating heatmaps and stats.
  • 2D: pitch rendering, density heatmaps, timelines. Custom tool (adapted): annotation & report composer for coaches.
  • External DB: team match library. Auth: ✅ team roles (coach/player). Analytics ⏭️: usage per view.
  • Large-dataset rendering: a full season of event data explorable in one linked view.
